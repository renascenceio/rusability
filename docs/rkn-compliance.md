# Rusability — соответствие требованиям РКН (Роскомнадзор)

_Документ для внутреннего использования и подготовки обращения в РКН._
_Последнее обновление: 12.07.2026._

---

## 1. Транспортная безопасность и протоколы (обзор РКН, июнь 2026)

| Требование | Статус | Как обеспечено |
|---|---|---|
| Действующий TLS-сертификат доверенного УЦ, без ошибок браузера | ✅ | Автоматически выпускается и продлевается на edge-слое Vercel |
| **Минимум TLS 1.2, TLS 1.3 включён** | ✅ | Согласуется TLS 1.3 с откатом на 1.2 |
| Устаревшие протоколы **отключены** — SSL 3.0, TLS 1.0, TLS 1.1 | ✅ | Не предлагаются сервером |
| Только современные наборы шифров | ✅ | Управляется на edge-слое |
| **HTTP/2 включён** (и HTTP/3) | ✅ | Включён по умолчанию на edge Vercel |
| Автоперенаправление HTTP → HTTPS | ✅ | 308-редирект на edge |

> Уточнение по протоколам: отключать нужно **старые** протоколы (SSL 3.0 / TLS 1.0 / 1.1). **TLS 1.3 остаётся включённым** — это рекомендованная версия.

## 2. HTTP-заголовки безопасности (реализовано в коде, `next.config.ts`)

Активируются на продакшн-деплое (`VERCEL_ENV=production`); проверяются через `curl -I https://rusability.ru` или SSL Labs.

| Заголовок | Значение |
|---|---|
| `Strict-Transport-Security` | `max-age=63072000; includeSubDomains; preload` |
| `Content-Security-Policy` | `upgrade-insecure-requests` |
| `X-Content-Type-Options` | `nosniff` |
| `X-Frame-Options` | `SAMEORIGIN` |
| `Referrer-Policy` | `strict-origin-when-cross-origin` |
| `Permissions-Policy` | `camera=(), microphone=(), geolocation=()` |

## 4. Доступность — ГОСТ Р 52872-2019 (аудит)

| Критерий ГОСТ | Статус | Примечание |
|---|---|---|
| Язык страницы объявлен (`<html lang="ru">`) | ✅ | `src/app/layout.tsx` |
| Ссылка «Перейти к основному содержимому» для клавиатуры | ✅ | Добавлена в `src/app/(public)/layout.tsx` |
| Альтернативный текст у изображений | ✅ | Полное покрытие `alt` у контентных изображений |
| Семантическая разметка (`header`, `main`, `nav`, `footer`) | ✅ | Используется во всех макетах |
| ARIA-роли и атрибуты для интерактивных элементов | ✅ | Кнопки, вкладки, переключатели снабжены `aria-*` |
| Масштабирование до 200% без потери контента | ✅ | Вёрстка на `rem`/`clamp`, адаптивная сетка |
| Контраст текста ≥ 4.5:1 | ⚠️ | Основной текст соответствует; **требуется финальная проверка** второстепенных серых подписей на светлой теме |
| Тёмная/светлая тема | ✅ | Переключатель тем на всех страницах |

**Остаточная задача по доступности:** прогнать контраст второстепенных подписей (`--muted-foreground`) через инструмент проверки (например, WAVE / axe) и при необходимости затемнить на один шаг.

## 5. Локализация данных (152-ФЗ) — ТРЕБУЕТ РЕШЕНИЯ

Это **единственное требование, не закрываемое кодом**.

- Сейчас приложение работает на глобальной инфраструктуре Vercel (серверы вне РФ).
- Сайт собирает персональные данные граждан РФ (регистрация, подписка на рассылку), поэтому по 152-ФЗ первичная запись ПДн должна выполняться на серверах, физически расположенных в РФ.

**Варианты:**
1. Перенести основное хранилище ПДн (БД) и обработку форм на хостинг/CDN в РФ (например, Yandex Cloud, VK Cloud, Selectel, Reg.ru).
2. Организовать зеркальную запись ПДн в РФ-хранилище (primary-в-РФ, реплика — для остального).
3. Уведомить РКН о месте нахождения БД ПДн (реестр операторов ПДн).

**Рекомендация:** обсудить и выбрать РФ-хостинг для БД/форм до обращения в РКН — автоматические проверки чаще всего фиксируют именно это.

## 6. Идентификация администратора домена через Госуслуги/ЕСИА

- Обязательно для доменов `.ru`/`.рф` **с 1 сентября 2026 г.**
- Выполняется у регистратора домена, не в приложении.
- **Действие:** подтвердить владельца домена rusability.ru через ЕСИА у регистратора.

## Что можно написать в РКН (сводка сделанного)

1. Настроены TLS 1.2/1.3, отключены SSL 3.0 и TLS 1.0/1.1, включены HTTP/2 и HTTP/3, действует автоперенаправление на HTTPS.
2. Подключены HTTP-заголовки безопасности: HSTS с preload, защита от MIME-sniffing, кликджекинга, корректная политика Referrer и Permissions.
3. Реализована контентная модерация: автофильтр запрещённых тем при публикации, автоблокировка после 3 нарушений, ручная блокировка администратором, ручная проверка новостей перед публикацией, реестр заблокированных пользователей.
4. Обеспечена доступность по ГОСТ Р 52872-2019: язык страницы, skip-ссылка, alt-тексты, семантическая разметка, ARIA, масштабирование 200%.

### Открытые вопросы (до отправки в РКН)
- Локализация ПДн на серверах в РФ (152-ФЗ) — выбрать РФ-хостинг.
- Идентификация владельца домена через ЕСИА (срок — 01.09.2026).
- Финальная проверка контраста второстепенного текста.
